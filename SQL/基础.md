# 事务
**简介：**==事务==是一组操作的集合，它是一个不可分割的工作单位，事务会把所有的操作视为一个整体一起向系统提交或撤销操作请求，即这些操作==要么同时成功，要么同时失败==

第一种事务控制方式：关闭事务的自动提交，再通过COMMIT提交事务和ROLLBACK回滚事务
![[Pasted image 20251223191442.png]]
第二种事务控制方式：通过START TRANSACTION或BEGIN来显式的开启事务，再看事务是否执行成功。如果执行成功，再执行COMMIT，反之则执行ROLLBACK。
